Introduction

This page is the report of the Path_planning project from Autorob course. Basically, it will talk about the motivation of using searching algorithms in the robot world. Then, it will show the principle of the four graph searching algorithms, listed as Depth-First Searching, Breadth-First Searching, Astar and Greedy-Best Searching, and the experiemental results in different configurations.

Motivation

Imagine our robot sits in a map and it wants to go somewhere else to perform a task. Here, the robot has no clue which route it should take, and it is important for us to plan ahead before actually moving. We assume the map is stationary, thus the robot can follow the route planned ahead to reach the goal.

Methodology

To search the world map, a common way to represent the world as grid map. With a certain resolution, the continous world map is represented as discrete grids or so called nodes. Each node has its neighbors and they are directly connected by edges. The robot can move to an adjacent node with a single step. For a 2D world, the grid map can be mathematically represented as a a 2D array. It is depend on us to decide which nodes are "neighbors" for a certain node, conventionally the directly connected 4 or 8 nodes.

Graph search algorithms explore a graph either for general discovery or explicit search. These algorithms carve paths through the graph, but there is no expectation that those paths are computationally optimal.Pathfinding algorithms build on top of graph search algorithms and explore routes between nodes, starting at one node and traversing through relationships until the destination has been reached.

Depth-Frist Search

A depth-first search (DFS) is an algorithm for traversing a finite graph. DFS visits the child vertices before visiting the sibling vertices; that is, it traverses the depth of any particular path before exploring its breadth. A stack is generally used when implementing the algorithm.

Breadth-Frist Search

BFS is a traversing algorithm where you should start traversing from a selected node (source or starting node) and traverse the graph layerwise thus exploring the neighbour nodes. You must then move towards the next-level neighbour nodes.

A-star Search

A* is an informed search algorithm, or a best-first search, meaning that it is formulated in terms of weighted graphs: starting from a specific starting node of a graph, it aims to find a path to the given goal node having the smallest cost (least distance travelled, shortest time, etc.). It does this by maintaining a tree of paths originating at the start node and extending those paths one edge at a time until its termination criterion is satisfied.

Results & Conclusion

To summarize the experimental results of each searching algorithms in various configurations, we use a table to display the path found in each map and the rank of time reflected by the number visited nodes

From the results, we can see that pattern of visited nodes and the final path fall in the expectation.

Depth first search pop the most recent node in the visit queue to explore, thus we can see that it traverses a tree structure by exploring as far as possible down each branch before backtracking.

Breadth First search dequeue the least recent node to explore, therefore we can see that it traverses a tree structure by fanning out to explore the nearest neighbors and then their sublevel neighbors.In this way, the final path found by BFS is optimal

Astar explores the node with smallest fscore, and the resulting tree expands towards the goal with the direction of heuristics. Therefore, it explores much less node compared to DFS ad BFS. Also, it guarantees the final path is optimal

Greedy Best-First Search expands the searching tree directly towards the goal. If blocked by a bugtrap or a wall, it will keep searching the nearest nodes around the block point until it finds a way out. In this method, we can see that the searching speed is extremely fast if there are few obstacles in the way to the goal point.
